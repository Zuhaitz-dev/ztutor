Under the hood of the preprocessor, the C99 `<stdbool.h>` header maps the friendly name `bool` directly to a core, built-in compiler keyword token called `_Bool`. The native `_Bool` implementation is an atomic primitive type engineered to hold exclusively the binary bits 0 and 1; if you try to assign any arbitrary non-zero integer (like 42 or -10) to a `_Bool` variable, the compiler generates instructions to clamp it automatically down to a clean 1.
---
In older legacy C89 projects, developers ran into severe integration collisions when blending third-party software frameworks. If Library A mapped its boolean model as `typedef char BOOL;` with `#define TRUE 1`, while Library B declared `typedef int BOOL;` with `#define TRUE -1` (a pattern frequent in retro Windows/COM APIs), compiling both headers into a single module triggered immediate compiler redefinition errors or silent bitwise masking bugs.
---
Even though a boolean value logically represents an individual single bit of information, modern hardware cannot directly read or write single isolated bits from raw system memory chips. To align with CPU data bus routing rules, compilers pad a `bool` variable allocation to occupy a complete addressable block, causing `sizeof(bool)` to evaluate to exactly 1 byte (8 bits) on almost all modern computing setups.
